Voici la description du contenu de chaque image, organisée par son titre (nom de fichier), en s'appuyant uniquement sur les informations fournies dans les sources.
Descriptions des images
1.png et 2.png Ces images montrent la page de téléchargement ou de support de VMware Workstation Pro (For Windows). La version affichée est 17.6 avec le niveau de Release Level Info 522389. Le nom du fichier de téléchargement est VMware-workstation-full-17.6.0-24238078.exe (447.97 MB). La date de publication (Release Date) est le Sep 03, 2024, et la date de dernière mise à jour (Last Updated) est le Aug 28, 2024.
3.png Cette image présente une vue de l'Explorateur de fichiers dans le dossier Downloads. Le fichier exécutable VMware-workstation-full-17.6.0-24238078.exe est sélectionné, affichant une taille de 447 MB.
4.png Il s'agit de l'écran de bienvenue du VMware Workstation Pro Setup Wizard (Assistant d'installation de VMware Workstation Pro), indiquant la version 17.
5.png Cette image montre l'écran du End-User License Agreement (Contrat de licence utilisateur final) de VMware Workstation Pro Setup, où l'utilisateur confirme l'acceptation des termes.
6.png L'installation de VMware Workstation Pro est en cours, et le statut indique spécifiquement l'installation des pilotes de réseau virtuel (Installing virtual network drivers).
7.png Cet écran confirme que l'assistant d'installation de VMware Workstation Pro a été complété (Completed the VMware Workstation Pro Setup Wizard).
8.png C'est l'interface principale de VMware Workstation Pro 17, présentant trois options principales : Create a New Virtual Machine (Créer une nouvelle machine virtuelle), Open a Virtual Machine (Ouvrir une machine virtuelle) et Connect to a Remote Server (Se connecter à un serveur distant).
9.png L'assistant de nouvelle machine virtuelle (New Virtual Machine Wizard) est lancé, et l'option Custom (advanced) (Personnalisée) est sélectionnée pour créer une machine virtuelle avec des options avancées.
10.png Cet écran de l'assistant permet de Choose the Virtual Machine Hardware Compatibility (Choisir la compatibilité matérielle de la machine virtuelle). La configuration choisie est Workstation 17.5 or later. Les limitations pour cette compatibilité incluent 128 GB de mémoire et 32 processeurs.
11.png Étape Guest Operating System Installation (Installation du système d'exploitation invité). L'option d'installation à partir d'un Installer disc image file (iso) est sélectionnée, mais le système d'exploitation n'a pas pu être détecté dans le chemin affiché.
12.png À l'étape Guest Operating System Installation, un fichier ISO (D:\Downloads\WIN10.PRO.AIO.U21.X64.(WPE).ISO) est sélectionné. Windows 10 x64 est détecté, et l'assistant note que ce système d'exploitation utilisera Easy Install.
13.png L'écran Easy Install Information est affiché pour l'installation de Windows 10 x64. Il demande la clé de produit et permet de choisir la Version of Windows to install (Windows 10 Education est sélectionné). La personnalisation est configurée avec le nom complet admin, un mot de passe et l'option de connexion automatique activée (Log on automatically).
14.png Cette étape permet de Name the Virtual Machine (Nommer la machine virtuelle). Le nom choisi est Windows 10 x64, et l'emplacement de sauvegarde est affiché.
15.png Il s'agit de la Processor Configuration (Configuration du processeur). Le nombre de processeurs est réglé sur 2, avec 1 cœur par processeur, pour un total de 2 cœurs de processeur.
16.png L'écran Memory for the Virtual Machine (Mémoire pour la machine virtuelle) est présenté. La mémoire allouée est de 2048 MB, ce qui correspond à la mémoire recommandée pour le système d'exploitation invité (Guest OS recommended minimum). La mémoire maximale recommandée est de 20.8 GB.
17.png Cet écran concerne le Network Type (Type de réseau) à ajouter. L'option sélectionnée est Use network address translation (NAT).
18.png Similaire à l'image précédente, cet écran montre l'option Use host-only networking sélectionnée, qui connecte l'OS invité à un réseau virtuel privé sur l'ordinateur hôte.
19.png L'utilisateur est invité à Select I/O Controller Types (Sélectionner les types de contrôleurs E/S) pour les disques virtuels SCSI. Le contrôleur LSI Logic SAS est sélectionné et recommandé.
20.png L'étape Select a Disk Type (Sélectionner un type de disque) est affichée. Le type de disque virtuel NVMe est sélectionné et recommandé.
21.png À l'étape Select a Disk (Sélectionner un disque), l'option choisie est Create a new virtual disk (Créer un nouveau disque virtuel).
22.png L'écran Specify Disk Capacity (Spécifier la capacité du disque) est affiché. La taille maximale du disque est de 60.0 GB (également la taille recommandée pour Windows 10 x64). L'option Allocate all disk space now (Allouer tout l'espace disque maintenant) est cochée, et le disque est configuré pour être Split virtual disk into multiple files (Divisé en plusieurs fichiers).
23.png L'écran Specify Disk File (Spécifier le fichier disque) indique que le disque virtuel de 60 GB sera créé en utilisant plusieurs fichiers disques, et le nom du fichier sera Windows 10 x64.vmdk.
24.png C'est l'écran final Ready to Create Virtual Machine (Prêt à créer la machine virtuelle), qui récapitule les paramètres choisis : Nom: Windows 10 x64, Disque dur: 60 GB, Split, Pre-allocated, Mémoire: 2048 MB, Adaptateur Réseau: Host-only, Version: Workstation 17.5 ou ultérieure. L'option Power on this virtual machine after creation (Démarrer cette machine virtuelle après sa création) est cochée.
25.png Une fenêtre d'état de VMware Workstation indique que l'application est en train de créer le disque (Creating Disk...).
26.png Un avertissement de VMware Workstation est affiché, informant l'utilisateur que la machine virtuelle fonctionne avec des mesures d'atténuation de canal latéral (side channel mitigations) activées, ce qui améliore la sécurité mais réduit également les performances.
27.png Un écran Windows Setup est affiché, permettant de Select the operating system you want to install. Une liste de différentes versions de Windows 10 (PRO - COMPACT, SUPERLITE, etc.) en architecture x64 est visible.
28.png L'écran Installing Windows du Windows Setup montre le début du processus d'installation avec le statut Copying Windows files (0%).
29.png L'interface de VMware Workstation affiche la machine virtuelle Windows 10 x64 en cours d'installation, avec l'écran Windows affichant le message Getting ready (Préparation).
30.png L'écran de la machine virtuelle affiche un message d'attente sur fond bleu : "This might take several minutes" (Cela pourrait prendre plusieurs minutes).
31.png L'écran de connexion de Windows 10 dans la machine virtuelle est affiché, avec l'utilisateur admin.
32.png Le bureau de Windows 10 x64 est visible dans l'interface de VMware Workstation, indiquant que l'installation du système d'exploitation est terminée et que le système est fonctionnel.
33.png Un menu contextuel de la machine virtuelle Windows 10 x64 est affiché dans la librairie de VMware Workstation, listant des informations résumées telles que la RAM (2 GB), le disque dur (60 GB Preallocated) et l'adaptateur réseau (Host-only).
34.png Il s'agit de la fenêtre Virtual Machine Settings (Paramètres de la machine virtuelle), affichant le résumé du matériel : 2 GB de Mémoire, 2 Processeurs, Disque dur (NVMe) de 60 GB (Preallocated) et un Adaptateur Réseau Host-only. Elle confirme que le lecteur CD/DVD est connecté à un fichier image ISO.
